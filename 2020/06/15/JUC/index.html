什么是JUC

进程、线程

进程：什么是进程？如打开一个应用程序，在后台运行的qq.exe就是一个程序、一个进程至少包含一个线程

线程：线程是在一个进程中有多条线程进行处理数据

概念理解—>并发、并行

  1. 并发：cpu的一核 分别取执行多条线程，底层实现原理（jvm的pc寄存器）

    通俗解释：并发是两个队列交替使用一台咖啡机

  2. 并行：多个线程或者进程同时进行工作

    通俗解释：并行是两个队列同时使用两台咖啡机

二、锁

传统锁 synchronized 和Lock锁

  1. 非静态锁，锁的为对象，对象可以有多个
  2. 静态锁，锁的为类，类只能有一个
> 区别

 1. Synchronized 内置的Java关键字， Lock 是一个Java类

      2. Synchronized 无法判断获取锁的状态，Lock 可以判断是否获取到了锁
               3. Synchronized 会自动释放锁，lock 必须要手动释放锁！如果不释放锁，死锁
                 4. Synchronized 线程 1（获得锁，阻塞）、线程2（等待，傻傻的等）；Lock锁就不一定会等待下去；
           5. Synchronized     可重入锁，不可以中断的，非公平；Lock ，可重入锁，可以  判断锁，非公平（可以自己设置）；
                6. Synchronized     适合锁少量的代码同步问题，Lock 适合锁大量的同步代码！

公平锁和非公平锁

公平锁：资源加载非常公平，不会插队。

非公平锁：资源加载可能会插队，如前线程执行需要3h后只需3s这样就会让后方先执行

默认锁为非公平

四、Callable

Callable –>区别Runnable

  1. Callable 可以返回值，能抛出被检查的异常。
  2. 基于Runnable的实例化类(FutureTask)适配器进行实现
  3. 有缓存，执行效率高

使用

public class TestCallable {

    public static void main(String[] args) throws ExecutionException, InterruptedException {
        FutureTask<Integer> futureTask = new FutureTask<>(new MyThread());//创建适配器
        //lambad 方式
        //FutureTask<Integer> futureTask = new FutureTask<>(()->{return 1;});//创建适配器
        new Thread(futureTask).start();
        //获取执行的返回值
        Integer integer = (Integer) futureTask.get(); //这个get  方法可能会产生阻塞！把他放到 最后

        System.out.println(integer);
    }
}
class MyThread implements Callable<Integer> {
    @Override
    public Integer call() throws Exception {
        System.out.println("执行");
        return 1024;
    }
}

一、CountDownLatch 计数器

/**
     * CountDownLatch ：使用案例--> 当（计数器）数达到某个特定数量的时候，那么会执行任务，否则进行阻塞等待
     * countDownLatch.countDown();//计数器数量减1
     * countDownLatch.await();//等待计数器归零，然后再向下执行
     */
    public void testCountDiwnLatch() throws InterruptedException {
        //当线程数为5时那么执行后面的任务,如果没有达成，那么会进行——阻塞等待
        CountDownLatch countDownLatch = new CountDownLatch(5);
        for (int i = 0; i <5 ; i++) {
            new Thread(()->{
                System.out.println("线程数到达");
                countDownLatch.countDown();//数量-1
            }).start();
        }

        countDownLatch.await();//等待计数器归零，然后再向下执行
        System.out.println("Close");
    }

二、CyclicBarrier 线程计数器

/**
    * CyclicBarrier:
    *  使用案例--> 当（线程数）达到某个特定数量的时候，那么会执行一个run 任务，否则进行阻塞等待
    *  每创建一个线程就换判断一次
    *
    */
   public void testCyclicBarrier()  {
       CyclicBarrier cyclicBarrier = new CyclicBarrier(5,()->{System.out.println("线程达到数量，执行了");});
       for (int i = 0; i <5 ; i++) {
           new Thread(()->{
               System.out.println("测试CyclicBarrier-->新增线程");
               try {
                   cyclicBarrier.await();//当没有达到线程数时，阻塞等待
               } catch (InterruptedException e) {
                   e.printStackTrace();
               } catch (BrokenBarrierException e) {
                   e.printStackTrace();
               }
           }).start();
       }
   }

三、Semaphore 阻塞队列

/**
    * Semaphore:阻塞队列
    *  new Semaphore(5)设置最大线程数
    *  semaphore.acquire();//拿到队列位置 相当于拿到锁
    *  semaphore.release();//释放队列资源 相当于释放锁,并唤醒等待的线程
    */
   public void testSemaphore(){
       Semaphore semaphore = new Semaphore(5);//定义队列最大线程5
       for (int i = 0; i <10 ; i++) {
           new Thread(()->{
               try {
                   semaphore.acquire();//拿到队列位置 相当于拿到锁
                   System.out.println(Thread.currentThread().getName()+"拿到了锁");
                   TimeUnit.SECONDS.sleep(2);
               } catch (Exception e) {
                   e.printStackTrace();
               }finally {
                   System.out.println(Thread.currentThread().getName()+"释放了锁");
                   semaphore.release();//释放锁
               }
           }).start();
       }
   }

七、线程池(重要)

什么是线程池？

  1. 因为创建线程时需要耗费时间和内存，所以存在了线程池
  2. 线程池是提前将线程资源创建好，如果要使用那么会直接来获取，如果使用完后会归还
  3. 提高了性能，时间，减少了内存的损耗

阿里巴巴官方手册不推荐使用工具类Executors工具类穿件 —>pool.TestUtilPool (了解)

1. 因为使用工具类创建可能会导致OOM

使用原生创建（重要）

7大参数

四大拒绝策略

理解 默认开启线程池是2个线程 当沾满时会在 阻塞队列中等待,如果阻塞队列满载时，会启用最大线程池

Runtime.getRuntime().availableProcessors();//获取当前cpu最大线程数（不要写固定值，要根据自己cpu来填写）

/**
 * 解析底层创建线程池
 *  7大参数
 *      new ThreadPoolExecutor(
 *                         2,//默认开启线程池
 *                         5,//最大线程池
 *                         5,//超时没有调用释放线程
 *                         TimeUnit.SECONDS,//时长单位
 *                         new ArrayBlockingQueue<>(2), //阻塞队列
 *                         Executors.defaultThreadFactory(),//工厂
 *                         new ThreadPoolExecutor.AbortPolicy()//拒绝策略
 *                 );
 *  4大拒绝策略
 *   new ThreadPoolExecutor.AbortPolicy() //当线程数超过线程池的最大时，那么抛出异常
 *   new ThreadPoolExecutor.CallerRunsPolicy()//当线程数超过线程池的最大时,哪里来回到哪里 将由调用此线程的线程执行，回调
 *   new ThreadPoolExecutor.DiscardPolicy()//线程池满了 那么就丢掉不执行
 *   new ThreadPoolExecutor.DiscardPolicy()//线程池满了 ，不报错，等待 尝试去抢第一个执行的线程，如果时间超过等待时长丢掉
 * 实例来理解
 *  ：理解 默认线程池是2个线程 当沾满时会在 阻塞队列中等待,如果阻塞队列满载时，会启用最大线程池
 *
 */
public class TestThreadPoolExecutor {
    public static void main(String[] args) {
        int max = Runtime.getRuntime().availableProcessors();//获取当前cpu最大线程数（）
        System.out.println(max);
        ThreadPoolExecutor threadPoolExecutor =
                new ThreadPoolExecutor(
                        2,//默认开启线程池
                        max,//最大线程池
                        2,//超时没有调用释放线程
                        TimeUnit.SECONDS,//阻塞时长单位
                        new LinkedBlockingQueue<>(3), //阻塞队列
                        Executors.defaultThreadFactory(),//默认工厂
                        new ThreadPoolExecutor.DiscardOldestPolicy()//拒绝策略
                );
        for (int i = 0; i <15 ; i++) {
            threadPoolExecutor.execute(()->{
                try {
                    System.out.println("创建"+Thread.currentThread().getName());
                    TimeUnit.SECONDS.sleep(2);
                }catch (Exception e){
                    System.out.println(e.getMessage());
                }finally {
                    threadPoolExecutor.shutdown();
                    //System.out.println("释放了"+Thread.currentThread().getName());
                }
            });
        }

    }

}

八、Stream

什么是Stream

  1. Stream 使用一种类似用 SQL 语句从数据库查询数据的直观方式来提供一种对 Java 集合运算和表达的高阶抽象。
  2. Stream API可以极大提高Java程序员的生产力，让程序员写出高效率、干净、简洁的代码。

使用 —>更多使用查看jdk文档

/**
 *	题目要求：一分钟内完成此题，只能用一行代码实现！
 *	现在有5个用户！筛选：
 *	1、ID  必须是偶数
 *	2、年龄必须大于23岁
 *	3、用户名转为大写字母
 *	4、用户名字母倒着排序
 *	5、只输出一个用户！
 */
public class TestStream {
    public static void main(String[] args) {
        List<User> list = new ArrayList<>();
        list.add(new User("A",20,10));
        list.add(new User("B",25,8));
        list.add(new User("C",26,7));
        list.add(new User("D",30,5));
        list.stream()
                .filter((u)->{return u.getId()%2==0; })//过滤
                .filter((u)->u.getAge()>23)//过滤率
                .map((t)->{return t.getName().toUpperCase();})//根据条件转换大写后，映射
                .sorted((t1,t2)->{return t1.compareTo(t2);})//进行比较
                .limit(1)//限流
                .forEach(System.out::println);//输出
        ;
    }
}

十、volatile（重要）

请你谈谈你对 Volatile 的理解

​ Volatile 是 Java 虚拟机提供轻量级的同步机制

​ 1、保证可见性

​ 2、不保证原子性

​ 3、禁止指令重排(因为有cpu内存屏障，所有禁止指令重排)

​

可见性

:在多线程中开辟新的线程，会进行划分一个新的线程工作区内存，内存需要调用主存中的数据，当某个线程在调用时会及时（实时）的读取主存中的数据，并且在释放锁后及时的更新数据，防止并发其他线程拿到脏数据

禁止指令重排

指令重排：java程序在运行时需要转换为字节码运行，字节码再转换为电脑语令运行，所以在运行时不一定是按照代码的上下顺序来执行的

原子性

:执行一个操作时，要么成功，要么不成功，原子性就包含了禁止指令重排

因为volatile非原子性，那么就要保证原子性，所以使用

  • java.util.concurrent.atomic 中提供好的类—>具体查看源码+api

十一、深入理解CAS

什么是CAS?

在计算机科学中，比较和交换（Conmpare And Swap）是用于实现多线程(同步)的原子指令。

CAS也就是底层同步的原理

CAS ： 比较当前工作内存中的值和主内存中的值，如果这个值是期望的，那么则执行操作！如果不是就一直循环（自旋锁）！

缺点：

1、 循环会耗时

2、一次性只能保证一个共享变量的原子性

3、ABA问题

代码举例说明ABA问题（相对于乐观锁）（狸猫换太子）

/**
 * 实例说明CAS
 *   也就是说：在运行过程中可能有其他线程对某条数据对其作出了改变，
 *              但是最终我想要获取的是最初的那个版本的值（这就出现了ABA问题）
 */
public class TestCAS {
    public static void main(String[] args) {
        test();
        test2();
    }
    //没有解决ABA问题
    public static void test(){
        AtomicReference<Integer> objectAtomicReference = new AtomicReference<>(10);

        //捣乱的线程
        objectAtomicReference.compareAndSet(10,15);
        objectAtomicReference.compareAndSet(15,11);

        boolean b = objectAtomicReference.compareAndSet(11, 15);//达到预期值进行替换改变
        System.out.println(b);
    }
    //解决ABA问题 添加版本号 使用原子类
    public static void test2(){
        AtomicStampedReference<Integer> objectAtomicStampedReference =
                new AtomicStampedReference<>(10,1);//初始值，版本号
        int stamp = objectAtomicStampedReference.getStamp();//获取版本号
        //达到预期进行替换改变（预期值，改变值，预期号，新的号）
        objectAtomicStampedReference.compareAndSet(10,15,objectAtomicStampedReference.getStamp(),objectAtomicStampedReference.getStamp()+1);
        objectAtomicStampedReference.compareAndSet(15,10,objectAtomicStampedReference.getStamp(),objectAtomicStampedReference.getStamp()+1);

        boolean b = objectAtomicStampedReference.compareAndSet(10, 15, stamp, objectAtomicStampedReference.getStamp() + 1);
        System.out.println(b);

    }
}
